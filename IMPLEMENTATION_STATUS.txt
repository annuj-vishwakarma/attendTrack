🚀 HOW IT WORKS
─────────────────────────────────────────────────────────────────────────────

1. App Initialization
   ├─ ThemeProvider wraps entire app
   ├─ useTheme hook initializes
   ├─ System preference detected (or saved pref loaded)
   └─ CSS variables applied to :root

2. Theme Storage
   ├─ User preference → localStorage
   ├─ Key: 'theme-preference'
   ├─ Values: 'dark' or 'light'
   └─ Auto-loaded on next visit

3. Theme Toggle
   ├─ User clicks toggle button
   ├─ setMode() updates React state
   ├─ applyTheme() updates all CSS variables
   ├─ All components update instantly (no re-render)
   └─ New preference saved to localStorage

4. Component Updates
   ├─ All styles read var(--color-*)
   ├─ CSS handles variable substitution
   ├─ 0.3s transition applied to body
   └─ Smooth visual change

📋 USER GUIDE
─────────────────────────────────────────────────────────────────────────────

FOR END USERS:
1. Open Settings (⚙️) [Admin users only]
2. Scroll to "Appearance" section
3. Click theme toggle button
4. Choose "Light" or "Dark" mode
5. All UI updates instantly
6. Preference auto-saves

FOR DEVELOPERS:
1. Use CSS variables for all colors
2. Import useTheme if you need theme state
3. Test new components in both modes
4. Reference THEME_COLOR_REFERENCE.md for colors
5. Follow naming convention: var(--color-[category]-[variant])
